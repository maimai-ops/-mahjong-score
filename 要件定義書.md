# 麻雀点数計算アプリ 要件定義書

作成日: 2026-07-04 / 版: v1.0

## 1. 概要

スマホのブラウザで動くWebアプリ。和了した手牌を写真に撮ると、AIが牌を認識し、いくつかの質問に答えるだけで正確な点数(翻・符・点)と役の内訳を表示する。

## 2. 利用者・利用シーン

- 対局中の点数確認: 実際の対局で素早く正確な点数を知りたい人
- 学習用途: 初心者が「なぜこの点数か」を役・符の内訳と解説で理解する

結果画面は常に「点数 + 内訳」を表示し、内訳をタップすると解説が開く構成で両立する。

## 3. 画面フロー

1. **撮影画面**: カメラで手牌14枚を撮影(またはアルバムから選択)
2. **認識中**: 写真をサーバー関数経由でAI認識APIに送信(数秒)
3. **確認・訂正画面**: 認識された14枚をグリッド表示。誤りは牌をタップ→牌ピッカーで修正。枚数不足/過剰も警告
4. **質問画面**: 点数計算に必要な情報を順に質問(下記 4.3)
5. **結果画面**: 点数、翻数・符、役の内訳、支払い(ツモ時の親子別)、解説

## 4. 機能要件

### 4.1 牌認識

- 入力: スマホカメラ写真1枚。和了時に倒して公開した手牌(+鳴き牌)をそのまま撮る。並べ直しは不要。ガイド枠は「手牌が画面に収まるように」の補助表示のみ
- 卓全体の撮影は非推奨(牌が小さく写り誤認識が増えるため)。手牌に寄って撮ることをUI上で促す
- ドラ表示牌が写り込んでいれば自動読み取りし、該当の質問をスキップ
- 認識: サーバー関数がClaude vision APIに画像を送り、牌34種×枚数のJSONを受け取る
- 赤5(赤ドラ)は通常の5と区別して認識
- 認識失敗・枚数不一致時は訂正画面で手動補完できること(フォールバックとしてフル手入力も可能)

### 4.2 訂正UI

- 14枚をグリッド表示、タップで牌ピッカー(萬子/筒子/索子/字牌タブ)を開いて差し替え
- 同一牌5枚以上などの不正状態を即時バリデーション

### 4.3 質問フロー(5〜7問)

1. 和了牌はどれか(14枚から1枚タップ)
2. ツモ or ロン
3. 親 or 子 + 自風・場風(1画面にまとめる)
4. リーチ状況(なし / リーチ / ダブルリーチ)+ 一発の有無
5. ドラ表示牌(牌ピッカーで選択、裏ドラ含む。リーチ時のみ裏ドラを聞く)
6. 鳴き(ポン・チー・カン)の有無と対象面子(門前ならスキップ)
7. 特殊状況(海底/河底/嶺上/槍槓)— 「なし」がデフォルトの1タップ

写真と回答から自動判定できる項目(平和・タンヤオ等の成立)は質問しない。

### 4.4 点数計算エンジン

- 標準リーチ麻雀フル対応: 1翻役〜役満(ダブル役満含む)、符計算、ドラ・赤ドラ・裏ドラ
- 親/子、ツモ/ロンの正確な点数と支払い内訳
- 高点法(複数解釈があるとき最高点を採用)
- 役なし(和了不可)や形式エラーの検出とメッセージ表示

### 4.5 結果表示

- 点数(例: 満貫 8000点)、翻・符、成立役の一覧と各翻数
- 符の内訳(学習用に展開表示)
- 各役タップで短い解説を表示

## 5. 非機能要件

- 対応環境: iOS Safari / Android Chrome の現行版
- 認識レスポンス: 5秒以内目標
- APIキーはサーバー関数内に保持し、クライアントに露出しない
- 運用コスト: ホスティングは無料枠、AI認識は1回あたり数円以内
- 写真はサーバーに保存しない(認識に使ったら破棄)

## 6. 技術構成

- フロントエンド: 静的HTML/CSS/JS(単一ページ)。点数計算はクライアント側JSで実装
- サーバー: Cloudflare Workers(無料枠)。役割は「画像を受けてClaude vision APIに中継し、牌JSONを返す」のみ
- ホスティング: Cloudflare Pages または GitHub Pages
- 将来: PWA化(manifest + service worker)でホーム画面追加に対応

## 7. 開発フェーズ

- **Phase 1**: 点数計算エンジン + 手入力UI(=訂正UI)+ 質問フロー + 結果画面。写真なしで完動させる
- **Phase 2**: 撮影 → サーバー関数 → AI認識の接続。訂正画面に認識結果を流し込む
- **Phase 3**: PWA化、解説コンテンツ充実、UI磨き込み

Phase 1だけでも「手入力の点数計算機」として実用になる。認識部分の精度検証はPhase 2で実写真を使って行う。

## 8. 検討事項・ローカルルール(未決)

- 喰いタンの有無、赤ドラの枚数(0/3/4)、切り上げ満貫 → 設定画面でON/OFF(デフォルト: 喰いタンあり・赤3・切り上げなし)
- 暗槓は写真で裏向き2枚になるため認識困難 → 質問フローで補完する方針
- 本場(積み棒)の加算 → 任意入力欄として追加するか要判断
- 流し満貫・途中流局は対象外
